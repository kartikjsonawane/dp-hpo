Approximate Dynamic Programming
for Neural Network HPO

DP-HPO formulates hyperparameter optimisation as a finite-horizon MDP and solves it via Bellman's backward induction. Achieves 98.28% accuracy with only 10 evaluations — vs 108 for exhaustive grid search. Exact under conditional independence. Bounded otherwise: f* − fDP-HPO ≤ (d−1)·ε.

// THEORETICAL FRAMEWORK
MDP FORMULATION
Hyperparameter Optimisation as a Finite-Horizon MDP
State = partial configuration. Action = next dimension value. Reward = terminal validation accuracy. Horizon = d dimensions.
V*(sᵢ) = max_{a ∈ Hᵢ₊₁} V*(T(sᵢ, a))
ADP APPROXIMATION
Value Function via Completion Vector
Uncommitted dimensions filled with DEFAULTS reference vector. Reduces evaluations from O(Π|Hᵢ|) to O(Σ|Hᵢ|).
V̂(sᵢ, a) = f(h₁*, …, hᵢ*, a, DEFAULT_{i+2}, …, DEFAULT_d)
THEOREM 1 — OPTIMALITY GAP
Bounded Suboptimality
Exact when dimensions are conditionally independent (ε=0). Bounded in general case.
f* − f_DP-HPO ≤ (d − 1) · ε
PROPOSITION 1 — EVALUATION COUNT
10 Evaluations for 4-Dim MLP Space
DEFAULTS config cached and shared across all sweeps. Each dim contributes |Hᵢ|−1 fresh evaluations.
1 + (3−1) + (4−1) + (3−1) + (3−1) = 10
