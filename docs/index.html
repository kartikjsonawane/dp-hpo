// THEORETICAL FRAMEWORK
MDP FORMULATION
Hyperparameter Optimisation as a Finite-Horizon MDP
State = partial configuration (committed dims + ★ for uncommitted). Action = next dimension value. Reward = terminal validation accuracy. Horizon = d (one stage per dimension).
V*(sᵢ) = max_{a ∈ Hᵢ₊₁} V*(T(sᵢ, a)) [Bellman equation]
ADP APPROXIMATION
Value Function Approximation via Completion Vector
Instead of evaluating all reachable configs, DP-HPO approximates V*(sᵢ, a) by completing uncommitted dimensions with DEFAULTS — a fixed reference vector. This reduces evaluations from O(Π|Hᵢ|) to O(Σ|Hᵢ|).
V̂(sᵢ, a) = f(h₁*, …, hᵢ*, a, DEFAULT_{i+2}, …, DEFAULT_d)
THEOREM 1 — OPTIMALITY GAP
Bounded Suboptimality Under Interaction
When hyperparameter dimensions are conditionally independent (ε=0), DP-HPO is globally optimal. In the general case, the gap to the true optimum is bounded by the pairwise interaction strength.
f* − f_DP-HPO ≤ (d − 1) · ε
LEMMA 2 — IMPORTANCE-FIRST ORDERING
Optimal Dimension Ordering Minimises Expected Gap
Sort dimensions by decreasing marginal variance: Var[f(a, DEFAULT_{-i})]. This ordering minimises E[Δ] and is computed for free from the first-pass sweeps.
order = argsort(-Var[f(a, DEFAULT_{-i})]) for a ∈ Hᵢ
PROPOSITION 1 — EVALUATION COUNT
Exact Evaluation Budget
The DEFAULTS config is shared across all d sweeps and cached (trained once). Each dimension i contributes |Hᵢ|−1 fresh evaluations. Total: 1 + Σ(|Hᵢ|−1) = 10 for the standard 4-dim MLP space.
1 + (3−1) + (4−1) + (3−1) + (3−1) = 1 + 2 + 3 + 2 + 2 = 10

def dp_hpo(X_train, y_train, X_val, y_val, order=None):
    """
    DP-HPO: Approximate Dynamic Programming for HPO.

    Commits each hyperparameter dimension optimally via Bellman
    backward induction. Exact under conditional independence.
    Gap bound: f* - f_DP-HPO <= (d-1) * epsilon  [Theorem 1]
    Evaluations: 1 + sum(|H_i| - 1) = 10         [Proposition 1]
    """
    committed = list(DEFAULTS)          # initialise with reference vector
    memo      = {}                      # evaluation cache

    for i in (order or range(len(DIMS))):   # MDP stages
        best_val, best_acc = None, -1

        for v in VALS[i]:                    # ADP value function sweep
            config         = list(committed)
            config[i]      = v            # V̂(sᵢ, v) = f(committed, v, DEFAULTS[i+1:])
            acc            = _evaluate(config, X_train, y_train,
                                        X_val, y_val, memo)
            if acc > best_acc:
                best_val, best_acc = v, acc

        committed[i] = best_val           # commit decision (Bellman argmax)

    return best_acc, len(memo)            # accuracy + #evaluations
